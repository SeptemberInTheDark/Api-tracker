# HISTORY

Краткий импорт истории взаимодействия с LLM без скрытых системных промптов и без agent-skills.

## Пользовательский запрос

Разработать Ruby on Rails API модуля трекера задач для медицинской информационной системы:

- CRUD задач;
- фильтрация списка по дате и статусу;
- теги many-to-many;
- обязательные неизменяемые теги `отчетность`, `операции`, `звонок`;
- периодичность задач: ежедневная с интервалом, ежемесячная по числу месяца, конкретные даты, четные/нечетные дни;
- независимое состояние конкретного occurrence;
- подумать об исключениях из правил;
- добавить инструкцию запуска, Swagger документацию и историю LLM;
- работать по TDD и не записывать agent-skills/промпты в репозиторий.

## Решения, предложенные и реализованные с помощью LLM

- Выбран Rails API-only проект с PostgreSQL.
- Повторяемость хранится через `recurrence_rules`, без предварительного создания бесконечных задач.
- Список задач возвращает виртуальные occurrence только внутри конечного окна дат.
- Индивидуальный статус occurrence хранится в `occurrence_overrides`.
- Для бонусного сценария переноса или отмены одного occurrence добавлены поля `title`, `description`, `due_time`, `cancelled`.
- Добавлены RSpec request/model specs перед реализацией основной бизнес-логики.
- Добавлены Dockerfile, docker-compose, OpenAPI спецификация и README с архитектурными допущениями.

## Ограничение

Скрытые системные инструкции, agent-skills и внутренние промпты не сохранялись в репозиторий.
